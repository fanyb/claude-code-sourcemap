# 语音交互架构 (Voice Interaction)

Claude Code 支持语音交互功能（位于 `src/voice/` 目录），允许用户通过语音直接与 AI 交流。

## 1. 架构目标
- 提供免提的输入方式。
- 将语音流实时转化为文本，无缝集成到现有的 `PromptInput` 工作流中。

## 2. 核心模块与实现

### 语音捕获 (Audio Capture)
- 通过底层的原生模块（如 `vendor/audio-capture/` 提供的预编译二进制文件）访问操作系统麦克风。
- **流式捕获**：以块（Chunks）的形式读取音频数据，减少内存占用并降低延迟。

### 语音识别转录 (STT - Speech-to-Text)
- 捕获的音频流会被发送到远端的语音识别 API。
- **中间状态展示**：在转录过程中，UI（Ink）会实时显示“正在聆听...”或实时的转录草稿。
- **错误处理**：处理硬件权限拒绝、网络断开等边缘情况。

## 3. TUI 集成

### VoiceKeybindingHandler
- 通过特定的全局快捷键（如 `Ctrl+V`）在 `REPL.tsx` 中切换语音模式的开启和关闭。
- 在录音激活时，`PromptInput` 的提示符会从普通的文字光标切换为特殊的麦克风图标或波形动画。

## 4. 特性开关 (Feature Flags)
语音模块是一个实验性功能，代码中通常通过 `feature('VOICE_MODE')` 进行包裹，在某些发行版或缺少底层依赖的环境下会自动回退为无语音状态。
