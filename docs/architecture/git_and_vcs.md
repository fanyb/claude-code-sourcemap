# Git 深度集成与版本控制 (Git Integration)

Claude Code 与 Git 深度集成，这使其能够理解代码的历史、感知当前的修改，并自动化繁琐的代码提交和分支管理工作。

## 1. 差异感知 (Diff Awareness)

系统在每次询问前会执行以下操作：
- **实时检测**：自动获取当前工作树中未暂存（unstaged）和已暂存（staged）的差异 (`git diff`)。
- **差异排序**：优先考虑与当前用户提问或上下文相关的文件差异，将其优先注入到模型 Prompt 中。
- **Git 根目录探测**：自动识别项目的 Git 根目录，以便在多级子目录中也能正确执行 Git 命令。

## 2. 自动化提交流程

通过 `/commit` 和 `/commit-push-pr` 命令，Claude Code 实现了全自动的代码提交流：
- **自动生成 Commit Message**：AI 根据 `git diff` 的内容，遵循 Conventional Commits 规范生成高质量的提交信息。
- **归因分析 (Commit Attribution)**：系统会记录哪些修改是由 AI 提议的，并可选地在提交信息中包含相关的元数据。
- **安全性确认**：在提交前，系统会再次列出所有修改的文件供用户审查。

## 3. 工作流与隔离 (Worktrees)

- **Worktree 管理**：在进行大规模重构或分支切换时，系统可以利用 Git Worktrees 创建一个独立的工作区，从而不干扰用户当前的开发环境。
- **状态快照**：在执行大规模修改前，系统会自动记录当前的 Git 状态，以便在操作失败或用户撤销时进行恢复。

## 4. 冲突处理

当 AI 的修改与用户当前的修改发生冲突时：
- **冲突识别**：系统会通过解析 Git 的冲突标记来识别问题。
- **AI 辅助修复**：AI 可以被指派去解决特定的 Merge Conflict，它能够理解两个版本的语义差异并给出合并建议。
