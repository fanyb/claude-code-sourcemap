# 插件与扩展系统 (Plugins & Extensibility)

Claude Code 的高度可扩展性得益于其精巧的插件与技能（Skills）加载机制。

## 1. 核心概念：插件 (Plugins) vs. 技能 (Skills)

### 插件 (Plugins)
- **定义**：更深层的扩展，通常由 JavaScript/TypeScript 编写。
- **能力**：可以注册新的斜杠命令（`/command`）、添加自定义工具（Tools）或注入生命周期钩子（Hooks）。
- **加载机制**：通过 `loadPluginCommands.js` 扫描插件目录，利用动态 `import()` 加载插件模块。

### 技能 (Skills)
- **定义**：基于 Prompt 的原子能力，通常存放在 `~/.claude/skills/` 目录下。
- **能力**：它们是预定义的指令集，AI 可以调用这些“技能”来执行特定任务（如按照特定的代码规范重构）。
- **发现机制**：系统在每次启动或环境变化时，会自动搜索并更新“技能列表”，将其注入模型的工具定义中。

## 2. 插件发现与加载流程

1. **扫描阶段**：启动时，系统检查 `--plugin-dir` 或默认插件路径。
2. **验证阶段**：解析插件的 `package.json` 或清单文件，验证版本兼容性。
3. **注册阶段**：插件实例被实例化，并将其定义的 `commands`, `tools` 和 `hooks` 注册到全局单例（如 `COMMANDS` 列表）中。

## 3. 生命周期钩子 (Hooks)

插件可以通过钩子介入主流程的各个阶段：
- **Pre-Tool Use**: 在 AI 调用工具前执行（例如，对 Bash 命令进行安全扫描）。
- **Post-Tool Use**: 在工具执行完成后执行。
- **Session Start/End**: 在会话开始或结束时执行（例如，LSP 服务的启动与清理）。

## 4. MCP 作为一种动态扩展

Model Context Protocol (MCP) 也是一种特殊的扩展方式。相比于本地插件，MCP 允许连接到运行在不同进程或机器上的工具服务器。
- **动态发现**：Claude Code 可以实时连接到新的 MCP 服务器，并立即获取其提供的工具。
- **标准化**：遵循统一的 MCP 协议，使得各种外部工具（如数据库、搜索 API）可以无缝集成。
