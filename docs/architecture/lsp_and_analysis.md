# LSP 与代码分析架构 (LSP & Code Analysis)

Claude Code 的核心竞争力之一是其能够利用 Language Server Protocol (LSP) 进行语义级别的代码分析。这使得 AI 能够理解符号之间的关系、定义跳转以及精确的诊断信息。

## 1. LSP 管理器 (`services/lsp/manager.ts`)

系统内置了一个精密的 LSP 生命周期管理器：
- **按需加载**：系统会根据当前项目中的文件类型（如 `.ts`, `.py`, `.java`）动态发现并启动相应的语言服务器。
- **协议翻译**：将复杂的 LSP 消息（如 `textDocument/definition`, `textDocument/references`）转化为模型可用的结构化数据。
- **并发管理**：支持同时运行多个语言服务器，以适应多语言项目。

## 2. 核心功能

### 符号定义与跳转 (Definitions)
- **精准查找**：当 AI 需要编辑一个类或函数时，它可以先调用 LSP 获取该符号的确切定义位置。
- **上下文补充**：通过分析一个函数的引用情况，AI 可以评估修改该函数可能带来的副作用。

### 代码诊断 (Diagnostics)
- **实时错误检测**：LSP 提供的 Lint 错误、类型错误会被实时反馈给 AI。
- **自动修复**：AI 可以利用 LSP 的错误提示，在执行修改后立即验证代码是否仍然能够通过类型检查。

### 符号索引 (Indexing)
- **全局视野**：在大型项目中，系统通过 `codeIndexing.js` 对项目的符号进行初步索引，帮助模型在无需读取全部文件的情况下找到相关的逻辑块。

## 3. 增强型工具

### 代码搜索工具
- 相比于简单的 `grep`（文本匹配），LSP 增强的搜索工具支持按照“符号类型”进行搜索（如“查找所有调用了 `sendNotification` 的地方”）。

### 智能重构
- 通过 LSP 提供的重命名 (Rename) 和提取 (Extract) 能力，AI 可以执行比简单的字符串替换更可靠的重构操作。

## 4. 性能与资源平衡

- 为了防止 LSP 服务器占用过多内存，系统会根据终端资源状况和任务需求，动态调整 LSP 的活跃状态，并在会话结束或长时间空闲时自动关闭服务器。
