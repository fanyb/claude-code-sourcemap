# 查询生命周期架构 (Query Lifecycle)

`QueryEngine` 是 Claude Code 的核心调度中心，负责管理单次对话的完整生命周期。

## 核心流程图

```text
[用户输入] 
    |
    v
+-----------------------------+
| processUserInput()          | 识别斜杠命令 (/commit, /review 等)
+--------------+--------------+
               |
               v
+--------------+--------------+
| fetchSystemPromptParts()     | 组装 System Prompt (工具定义 + 上下文)
+--------------+--------------+
               |
               v
+--------------+--------------+
| query() 循环开始             | <--------------------------+
+--------------+--------------+                            |
               |                                           |
               v                                           |
+--------------+--------------+                            |
| 模型推理 (API Call)          |                            |
+--------------+--------------+                            |
               |                                           |
     +---------+---------+                                 |
     |                   |                                 |
[文字响应]           [工具调用请求]                         |
     |                   |                                 |
     v                   v                                 |
[结束循环]       +-------+-------+                         |
                 | 权限检查       |                         |
                 +-------+-------+                         |
                         |                                 |
                 +-------+-------+                         |
                 | 工具执行       |                         |
                 +-------+-------+                         |
                         |                                 |
                 +-------+-------+                         |
                 | 结果反馈 (User) | -----------------------+
                 +---------------+
```

## 关键类与职责

### 1. `QueryEngine`
- **持久化**：管理对话的消息历史 (`mutableMessages`)。
- **流式输出**：通过异步生成器输出 `SDKMessage`，允许 UI 实时更新进度。
- **中断处理**：持有 `AbortController`，支持用户随时停止 AI 思考。

### 2. `query.ts`
- **API 交互**：封装了对 Anthropic SDK 的调用。
- **重试机制**：处理网络抖动或模型过载。
- **Token 统计**：在每次交互后累加使用量和成本。

### 3. `processUserInput.ts`
- **预处理**：在消息发送给模型前，先处理本地命令。
- **附件管理**：处理用户通过 CLI 传入的文件路径，将其转化为模型可读的上下文。

## 状态转换
- **Thinking**: 模型正在生成响应。
- **ToolUsing**: 模型请求执行工具，系统正在处理（可能在等待用户授权）。
- **Compacting**: 当上下文过长时，系统自动触发压缩逻辑（Snip）。
