# 上下文与记忆管理架构 (Context & Memory)

Claude Code 的核心竞争力在于其对大型代码库的理解能力。为了在模型有限的上下文窗口中保持高效，它采用了一套精密的上下文筛选与压缩机制。

## 1. 上下文注入 (Context Injection)

系统在启动和每次提问时，会动态收集以下信息并注入到 Prompt 中：
- **项目结构**：自动运行 `ls -R` 或类似的目录遍历。
- **环境信息**：操作系统、Shell 类型、当前 Git 分支。
- **引导文件**：自动搜索并读取 `CLAUDE.md`（项目指令）和 `README.md`。
- **Git 状态**：当前的未提交更改 (`git diff`) 和最近的提交记录。

## 2. 消息压缩机制 (Compaction / Snip)

当对话历史过长，接近 Token 上限时，系统会触发自动压缩（由 `services/compact/compact.ts` 处理）：

### 压缩流程：
1. **内容清理**：从历史消息中移除大文件内容、图片数据和冗余的工具执行进度。
2. **摘要生成**：调用模型生成之前对话的“语义摘要”。
3. **边界划分**：在消息列表中插入一个 `SystemCompactBoundaryMessage`。
4. **状态保留**：压缩后的历史仅保留摘要，而新的对话从该边界开始。

### 微压缩 (Micro-compacting)：
对于非常频繁的工具调用结果，系统会进行“行级”或“字段级”的截断，只保留模型最需要的部分。

## 3. 记忆持久化

- **Session 恢复**：通过 `utils/conversationRecovery.ts` 将对话保存到本地缓存目录。即使 CLI 崩溃，用户也可以通过 `claude --resume` 恢复之前的上下文。
- **CLAUDE.md**：鼓励用户将长期规则（如代码风格、常用测试命令）写入该文件，作为 Agent 的“永久记忆”。

## 4. Token 估算与配额

系统内置了 `tokenEstimation.js`，在发送请求前预估 Token 消耗。如果预计会溢出，会提前提示用户手动压缩（`/compact`）或自动触发 `Snip`。
